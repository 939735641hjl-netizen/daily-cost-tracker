需要配置 Firebase

首次部署请按以下步骤操作（约 10 分钟）

第一步：创建 Firebase 项目

Firebase 是 Google 的免费后端平台，免费额度完全够个人使用。

1
打开 firebase.google.com，用 Google 账号登录，点击「创建项目」
2
输入项目名（例如 daily-cost），关闭 Google Analytics，点击「创建」
3
进入项目后，点击左侧「Firestore Database」→「创建数据库」→ 选「以生产模式启动」→「完成」
4
点击左侧「Authentication」→「开始」→「电子邮件地址/密码」→ 启用 → 保存

第二步：获取配置代码

在 Firebase 控制台获取你的项目配置。

1
点击左上角「项目设置」（齿轮图标）→「常规」→ 下滑到「您的应用」
2
点击「</>」（Web 应用）→ 输入应用名称 → 注册应用
3
复制出现的 firebaseConfig 对象（包含 apiKey 等 6 个字段）

第三步：修改 index.html

将配置填入代码顶部，然后重新推送到 GitHub。

1
打开 GitHub 仓库中的 index.html，找到以下位置
// ===== FIREBASE CONFIG ===== const FIREBASE_CONFIG = { apiKey: "YOUR_API_KEY", authDomain: "YOUR_PROJECT.firebaseapp.com", ...
2
把 YOUR_... 占位符替换成你从 Firebase 复制的真实值，保存提交

第四步：设置安全规则

在 Firestore 控制台 → 规则，替换为以下内容并发布：

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /users/{uid}/{doc=**} { allow read, write: if request.auth != null && request.auth.uid == uid; } } }
⚠️ 提示：Firebase 在中国大陆需要科学上网，建议在有网络条件时完成配置。

使用说明

什么是日均费用？

日均费用衡量一件物品每天"消耗"了多少钱。用得越久，日均越低。灵感来自有数 App 的资产摊销——把一次性大额消费分摊到每一天，帮你判断东西值不值、该不该换。

计算公式

日均 = 购入价格 ÷ max(今天 − 购买日期, 1天)
本月 = 日均 × 30　本年 = 日均 × 365

颜色含义

＜ ¥10 / 天 很划算，继续用下去
¥10 ~ 50 / 天 适中，正常消费水平
＞ ¥50 / 天 偏贵，考虑是否值得
